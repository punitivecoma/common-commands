Repair Commands

The sequence of running DISM /Online /Cleanup-Image /RestoreHealth followed by sfc /scannow creates a robust strategy for repairing Windows installations. It first fixes potential damage at the source before verifying and restoring the integrity of essential system files. This approach maximizes the chances of resolving errors and enhances system stability.

DISM Command

DISM /Online /Cleanup-Image /RestoreHealth

SFC Command

sfc /scannow

Networking Commands

The command prompt commands related to networking are powerful tools for managing, testing, and troubleshooting network configurations. Here’s a breakdown of some common network commands in Windows, along with their purposes:

1. IP Configuration

Purpose: Displays the current IP configuration for all network interfaces.

Usage: ipconfig /all shows detailed information including the IP address, subnet mask, and default gateway.

ipconfig /all

2. Flush DNS

Purpose: Clears the DNS resolver cache to resolve DNS-related issues.

Usage: ipconfig /flushdns clears the DNS cache, which can help fix problems accessing websites.

ipconfig /flushdns

3. Ping Command

Purpose: Tests connectivity between your computer and another network device.

Usage: ping [hostname/IP] sends packets to the specified address to check if it’s reachable.

ping google.com

4. Tracert Command

Purpose: Traces the route packets take to a specified destination.

Usage: tracert [hostname/IP] shows each hop along the route to the target.

tracert google.com

5. Netstat Command

Purpose: Displays active network connections and listening ports.

Usage: netstat -ano shows connections with associated Process IDs (PIDs).

netstat -a

6. Nslookup Command

Purpose: Queries DNS to obtain the domain name or IP address mapping.

Usage: nslookup [domain name] retrieves the IP address for a specified domain.

nslookup example.com

7. Route Command

Purpose: Displays and modifies the IP routing table.

Usage: route print shows current network routes; route add [destination] [mask] [gateway] creates a new route.

route print

8. ARP Command

Purpose: Displays and modifies the ARP (Address Resolution Protocol) cache.

Usage: arp -a shows the current ARP entries.

arp -a

9. Netsh Command

Purpose: A versatile command for managing network configurations.

Usage: netsh interface show interface displays the status of network interfaces.

netsh interface show interface

Troubleshooting Windows Update Errors

Use the following commands in the following order to troubleshoot issues related to Windows Updates. These commands stop Windows Update services, delete temporary files, and restore system files to ensure a smooth update process.

1. Stop Windows Update Services

net stop wuauserv

2. Stop Cryptographic Services

net stop cryptSvc

3. Stop Background Intelligent Transfer Service

net stop bits

4. Stop Windows Installer Service

net stop msiserver

5. Delete SoftwareDistribution Folder

del %windir%\SoftwareDistribution /s /q

6. Start Windows Update Services

net start wuauserv

7. Start Cryptographic Services

net start cryptSvc

8. Start Background Intelligent Transfer Service

net start bits

9. Start Windows Installer Service

net start msiserver

10. Repair Windows Image

DISM /Online /Cleanup-Image /RestoreHealth

11. Verify System Files

sfc /scannow

Built-in Windows Tools

Windows provides a variety of built-in tools for system management, troubleshooting, and configuration. Use the following commands to access these useful utilities:

System Information

Purpose: Displays detailed system information.

Usage: The systeminfo command provides detailed system specifications, including OS version, hardware configuration, network settings, installed updates, and boot time, aiding in diagnostics and troubleshooting.

systeminfo

IP Configuration

Purpose: Displays the current IP configuration for all network interfaces.

Usage: The ipconfig command displays detailed network configuration information, including IP addresses, subnet masks, and default gateways for all network interfaces.

ipconfig

List Running Processes

Purpose: Lists all currently running processes.

Usage: The tasklist command provides a list of currently running processes on the system, along with their process IDs (PIDs) and memory usage, aiding in process management and troubleshooting.

tasklist

User Accounts

Purpose: Displays all user accounts on the system.

Usage: The net user command displays a list of user accounts on the system, along with basic information about each account, useful for user management and account administration.

net user

Disk Usage

Purpose: Shows used and free disk space on all drives.

Usage: The wmic logicaldisk get size,freespace,caption command provides information on disk drives, showing total size and available free space for each logical disk on the system.

wmic logicaldisk get size,freespace,caption

Current Active Sessions

Purpose: Lists user sessions on the local machine.

Usage: The query user command lists active user sessions on the local machine, including session IDs, usernames, and session states, aiding in user management and monitoring.

query user

Device Manager

Purpose: Opens the Device Manager.

Usage: The devmgmt.msc command opens Device Manager, allowing users to view and manage hardware devices and their drivers on the system.

devmgmt.msc

Task Manager

Purpose: Opens the Task Manager.

Usage: The taskmgr command opens Task Manager, providing an overview of running applications, processes, and system performance, enabling users to manage tasks effectively.

taskmgr

Event Viewer

Purpose: Opens the Event Viewer.

Usage: The eventvwr command opens Event Viewer, which displays detailed logs of system events and errors, aiding in troubleshooting and system monitoring.

eventvwr

Control Panel

Purpose: Opens the Control Panel.

Usage: The control command opens the Control Panel, providing access to system settings and configuration options for managing various aspects of the Windows operating system.

control
